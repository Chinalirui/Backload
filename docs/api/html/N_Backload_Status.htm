Backload.Status Namespace

Contains implementations of interfaces in the Backload.Contracts.Status namespace like IFileStatus which provides a list of available IFileStatusItem files processed within the current request context.

The files list can be accessed at runtime in the FileStatus property.

Classes
  ClassDescription
BackloadResult
Base class for specialized types providing the results of the execution pipeline
CoreResult
Extends BackloadResult with a Object result usually used for a custom response to the client
FileDataResult
Extends BackloadResult with a byte array property for file data. Used when Backload is configured to handle downloads (e.g. thumbsUrlPattern="{Backload}"). Default behavior is direct file download.
FileStatus
Provides a list of files processed in the current request
FileStatusItem
Provides file meta data for a single file. This is the central interface to get information about a processed file in the current request
FileStatusResult
Extends BackloadResult with a property for file meta. This class provides the standard result object when a JSON result is requested. To return a complete customized JSON result object ICoreResult.
Remarks
 Important
Only for internal use. The classes in this namespace should not be used directly. At runtime you can access the file status in the FileStatus property.
 Tip
It is strongly recommended to use the inferfaces provided in the Backload.Contracts.Status namespace wherever possible instead of the implementations.
