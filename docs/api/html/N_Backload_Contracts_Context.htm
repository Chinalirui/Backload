Backload.Contracts.Context Namespace
Contains interfaces used for Backload's execution Backload.Contracts.Context The most important interfaces in this namespace is IBackloadContext, which provides access to most context related types. Another interface is IWebConfig which provides access the runtime configuration properties and settings of the static configuration (Web.Backload.Config).

IFileDataResult
Extends IBackloadResult with a byte array property for file data. Used when Backload is configured to handle downloads (e.g. thumbsUrlPattern="{Backload}"). Default behavior is direct file download.
IFileInfo
Provides basic file meta data and an FileInfo object if available. A derived class can be used to create a IFileStatusItem instance.
IFileStatusResult
Extends IBackloadResult with a property for file meta. This class provides the standard result object when a JSON result is requested. To return a complete customized JSON result object ICoreResult.
IImageInfo
ImageInfo provides image size and resolution when the current uploaded file is an image
IJsonResult
Base interface for all interfaces returning JSON data to the client IFileStatusResult and ICoreResult
IRequestValuesBag
Provides access to form and query values of the request. Backload specific values are available in BackloadValues
IStorageInfo
Creates and provides storage information for a IFileStatusItem object using the current storage context and configuration settings (Web.Backload.config)

RequestType
By default, an incoming GET request wants a list of files returned in JSON format. You can configure the component to return a file or a thumbnail of a file. For example, you may want to dynamically create thumbnails instead of storing them in a subfolder (Configuration example: thumbsUrlPattern="{Backload}"). If the GetRequestType is File or Thumbnail, the file data is stored in the first Status.Files item (bytes[] FileData property).
