Backload.Contracts.Context.Config Namespace
Contains interfaces for the runtime configuration of the XML-based configuration (web.Backload.config) file like IWebConfig.
Interfaces
  InterfaceDescription
IAzureBlobStorageConfig
Provides Azure Blob storage specific settings
IAzureFileStorageConfig
Provides Azure File storage specific settings
IBasicStorageCloudConfig
Provides cloud storage provider configuration settings
IBasicStorageConfig
Base interface for storage configuration
ICacheManagerConfig
Internal use only. Sets the cache headers
IClientFilesConfig
Enables access to the configured client side script and style paths. Used for ASP.NET server side bundling only.
ICloudConfig
Base interface for cloud storage provider interfaces
IContentTypeConfig
Interface for a specific content type element used in the ContentTypes configuration
IContentTypesConfig
Interface to define the content types feature
ICorsConfig
Configures CROSS-ORIGIN RESOURCE SHARING (CORS). CORS is used in environments, where file requests (e.g. GET) come from different domains (e.g. client: http://www.yourwebsite.com, Backload: http://fileservice.com/Backload/Filehandler).
IDatabaseConfig
Provides access to the values of the Database configuration
IFileSystemConfig
Provides access to the values of the FileSystem configuration
IGoogleDriveConfig
Provides Google Drive specific storage settings
IImagesConfig
Provides image configuration settings
IImagesConfigBase
Provides basic image configuration settings
ISecurityConfig
Provides security configuration settings
Remarks
Pro/Enterprise version only
IThumbnailsConfig
Provides thumbnail configuration settings
IWebConfig
Provides access the configuration of the Web.Backload.config file.

internally used, to set the data source for operations (e.g. BASE64 creation)
FileExistsType
Sets or gets the behavior if a uploaded file already exists (Unique file names feature is not set [default] or set to false)
FileNotFoundType
Sets or gets the behavior if a requested file is not found
PluginType
Specifies the client side plugin. Set it to a special plugin, to set the return data format for this plugin. Set it to Auto, if you send a query-string with the plugin name (plugin=[plugin-name]). Set it to Custom if you create the returned output in the OutgoingResponseCreated event or an IOutgoingResponse extension
ResizeModeType
Image and thumbnails resize mode
ReturnExtraInfoType
Used to return extra information, not defined in the default returned data format, to the client,
StorageAccessType
Defines cloud storage access types
StorageProviderType
Determines the storage provider
WatermarkPositionType
Vertical position of a watermark to be placed on an image
Remarks
 Tip
It is strongly recommended to use the inferfaces provided in the Backload.Contracts namespace wherever possible instead of the implementations.
