Backload root namespace
It is strongly recommended to use the inferfaces provided in the [Backload.Contracts] namespace wherever possible instead of using the implementations in the [Backload] namespace directly. Setup instructions can be found on the GitHub wiki pages.
Namespaces
NamespaceDescription
Backload

Root namespace for all sub-namespaces and types. This namespace contains the important FileHandler class implementing the IFileHandler interface. An instance of this type, created by calling the static Create method, is the central starting point for all further operations.

Backload.Configuration
Contains classes for reading and parsing the static XML-based configuration (web.Backload.config) file.
Backload.Context

Contains implementations of Backload's execution Backload.Contracts.Context related interfaces. The most important types in this namespace implement the IBackloadContext, which provides access to most context related types in its properties. Another type implements IWebConfig which provides access the runtime properties and settings of the static configuration (Web.Backload.Config).

Related interfaces and enumerations are available in the Backload.Contracts.Context namespace.

Backload.Context.Config
Contains implementations of the interfaces in Backload.Contracts.Context.Config namespace providing the runtime configuration of the XML-based configuration (web.Backload.config) file.
Backload.Context.DataProvider

Contains implementations of the interfaces in IBackloadDataProvider namespace providing a wrapper for the current HTTP context. BackloadDataProvider is internally used to access variables of the provided request context.

The Backload component accepts a IBackloadDataProvider a parameter for its Init(IBackloadDataProvider, IBackloadStorageProvider) method. You can create a new BackloadDataProvider instance from an HTTP context or by calling one of the available constructors. THis way it is possible to run the Backload component in a non web-based environment.

Backload.Context.Tracing
Contains the implementations of the ITraceManager interface providing tracing and logging support.
Backload.Contracts.Context
Contains interfaces used for Backload's execution Backload.Contracts.Context The most important interfaces in this namespace is IBackloadContext, which provides access to most context related types. Another interface is IWebConfig which provides access the runtime configuration properties and settings of the static configuration (Web.Backload.Config).
Backload.Contracts.Context.Config
Contains interfaces for the runtime configuration of the XML-based configuration (web.Backload.config) file like IWebConfig.
Backload.Contracts.Context.Provider

Contains the IBackloadDataProvider interface providing a wrapper for the current HTTP context.

The Backload component accepts a IBackloadDataProvider a parameter for its Init(IBackloadDataProvider, IBackloadStorageProvider) method. You can create a new BackloadDataProvider instance from an HTTP context or by calling one of the available constructors. This way it is possible to run the Backload component in a non web-based environment.

Backload.Contracts.Context.Tracing
Contains the ITraceManager interface providing tracing and logging support.
Backload.Contracts.Eventing
Contains the definitions for the IBackloadEvents and related interfaces for server side events.. providing tracing and logging support.
Backload.Contracts.Eventing.Params
This namespace contains the definitions for the IBackloadEventParamBase and derived interfaces providing event specific parameters to registered event handlers.
Backload.Contracts.FileHandler

This namespace contains the important IFileHandler interface which is implemented by the FileHandler class. An instance of this type, created by calling the static Create method, is the central starting point for all further operations.

Backload.Contracts.Imaging

Contains definition of the IImageResizer interface which provides image processing features like resizing, cropping, canvas, watermark or background. While the implementing class internal you can create an instance by calling the Create method on ImageResizer provided in Services.

Backload.Contracts.Plugin
This namespace contains definitions for supported client plugin handler interfaces. These interfaces are internally used to create a client plugin specific result object.
Backload.Contracts.Services
The interfaces of this namespace provide definitions for the execution pipeline, a high-level API defined in the IServiceHub interface, a low-level API defined in the ICoreServices interface and at the bottom level storage context specific services for file system, database and cloud storage only internally available
Backload.Contracts.Services.Database
The interfaces in this namespace can be implemented by an external class to provide storage services for the Backload component. While it is often implemented by a DBContext (Entity Framework) it is not limited to be used in a database context.
Backload.Contracts.Status

Contains important interfaces IFileStatus which provides a list of available IFileStatusItem files processed within the current request context.

The files list can be accessed at runtime in the FileStatus property.

Backload.Plugin
Contains implementations for the interfaces in the Backload.Contracts.Plugin namespace. Internally used to create a client plugin specific result object.
Backload.Plugin.Handler
Contains client plugin handler implementations for the interfaces in the Backload.Contracts.Plugin namespace. Internally used to create a client plugin specific result object.
Backload.Services
Contains implementations of the interfaces in the Backload.Contracts.Services namespace. The classes of this namespace provide the execution pipeline, a high-level API defined in the IServiceHub interface, a low-level API defined in the ICoreServices interface and at the bottom level storage context specific services for file system, database and cloud storage only internally available
Backload.Status

Contains implementations of interfaces in the Backload.Contracts.Status namespace like IFileStatus which provides a list of available IFileStatusItem files processed within the current request context.

The files list can be accessed at runtime in the FileStatus property.
