Backload.Context.Config Namespace
Contains implementations of the interfaces in Backload.Contracts.Context.Config namespace providing the runtime configuration of the XML-based configuration (web.Backload.config) file.
Classes
  ClassDescription
AzureBlobStorageConfig
Provides Azure Blob storage specific settings
AzureFileStorageConfig
Provides Azure File storage specific settings
BasicStorageCloudConfig
Provides cloud storage provider configuration settings
BasicStorageConfig
Provides access storage related configuration settings
CacheManagerConfig
Provides cache configuration
ClientFilesConfig
Enables access to the configured client side script and style paths. Used for ASP.NET server side bundling only.
ContentTypeConfig
Class for a specific content type element used in the ContentTypes configuration
ContentTypesConfig
Class to configure the content types feature
CorsConfig
Configures CROSS-ORIGIN RESOURCE SHARING (CORS). CORS is used in environments, where file requests (e.g. GET) come from different domains (e.g. client: http://www.yourwebsite.com, Backload: http://fileservice.com/Backload/Filehandler).
DatabaseConfig
Provides access to the values of the database configuration
FileSystemConfig
Provides access to the values of the FileSystem configuration
GoogleDriveConfig
Provides Azure Blob storage specific settings
ImagesConfig
Provides extended image configuration settings
ImagesConfigBase
Provides basic image configuration settings
LicenseConfig
Provides license configuration data
SecurityConfig
Provides security configuration settings
Remarks
Pro/Enterprise version only
ThumbnailsConfig
Provides thumbnail configuration settings
WebConfig
Provides access the configuration of the Web.Backload.config file.
Remarks
 Important
Only for internal use. The classes in this namespace should not be used directly. At runtime you can access the configuration in the related property of the Context Backload context.
 Tip
It is strongly recommended to use the inferfaces provided in the Backload.Contracts.Context.Config namespace wherever possible instead of the implementations.
