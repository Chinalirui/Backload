The AzureFileStorageConfig type exposes the following members.

Properties
  NameDescription
AccessType
StorageAccessType defines the cloud storage access types
AuxFilesFolder
Subfolder for supporting files (previews, mappings, etc). If "auxFilesFolder" is an empty string, supporting files will be stored in a subfolder within the upload folder. If "auxFilesFolder" is not empty (e.g. auxFilesFolder="_aux") supporting files will be stored in an external path ([filesRoot]\[auxFilesFolder]\). In this case the main upload directories only contain the uploaded files, which improves some processes like generating list of available files (GET) for large file lists. [Default: ""].
ChunkPathPrefix
Temporary folder prefix for file chunks.
Connection
Storage connection information
CopiesRoot
Path to store copies of the uploaded files if configured (e.g. ~/copies).
Extension
Name of the extension file. [Default: "Backload.Plugin.AzureFileStorage.dll"].
FilesRoot
Root storage location as absolute (UNC) path (Example: \\{yourstorage}.file.core.windows.net\{share}\ or z:\{share}\). Note: You must also set a the virtual path name in the "virtualDirectory" attribute [Default: "files"]
Initialized
True, if storage root folders have been created
MakeCopies
True if copies should be created (Pro/Enterprise)
ObjectsRoot
Sub folder within FilesRoot. Example you have several web applications which store files into a different sub folder (this is ObjectsRoot) of FilesRoot.
PathSeperator
Character used to separate path or directory levels. Default: '\'
ShareName
Name of the share. Must begin with a lowercase character [Default: "files"]
TokenExpires
Token access only. Time period (minutes) where accessing a resource is possible with a valid token. [Default: "60"]
VirtualDirectory
Name of an IIS virtual directory which points to the root or an absolute storage endpoint URL (https://{yourstorage}.file.core.windows.net/{share}/). If virtualDIrectory is the name of an IIS virtual directory, the download URL will be like http://yourwebserver/[appname]/{virtualDirectory}/{filename}. Otherwise the download URL will be the endpoint URL. In this case access to the file is only possible with a valid access token (see connectionString and accessType). Note: if virtualDirecotry is an empty string or a slash ("/") you can retrieve files only by the Backload component (filesUrlPattern="{backload]") [Default: "/"]
