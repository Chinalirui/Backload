The BackloadDataProvider type exposes the following members.

Properties
  NameDescription
ApplicationPath
Physical path to the application in the local file system, corresponds to HttpRequestBase .PhysicalApplicationPath
Examples
dataStore.FileHandlerUrl = @"d:\applications\myapp"
BackloadValues
IBackloadRequestValues provides access to external request parameters Backload uses to process the request
DisposeStream
If true, stream resources will be released after internal processing.
ExecutionContext
ExecutionContextType defines the current execution context. Web environment (Web application) or Computer (Desktop/Server application). Default: Web.
ExecutionMethod
ExecutionMethodType defines an execution method, whether the files in the current request should be retrieved (Get), stored (Store) or deleted (Delete). In an HTTP environment Get and Delete refers to the HTTP methods (GET and DELETE) and Store to the HTTP methods (POST and PUT).
Files
List of files to store or null in GET or DELETE requests
UserInfo
This property provides a list of IBackloadDataProviderFile files to store (Null in GET or DELETE requests).
WebContext
HttpContextBase provides a web HTTP context in current request is web based
WebRequest
IBackloadDataProviderRequest provides web request data in a web execution context
WebResponse
IBackloadDataProviderResponse provides web response values in a web execution context
