Backload.Contracts.Services.Database Namespace
The interfaces in this namespace can be implemented by an external class to provide storage services for the Backload component. While it is often implemented by a DBContext (Entity Framework) it is not limited to be used in a database context.
Classes
  ClassDescription
BackloadStorageProviderFile
Provides simple file data model usually used in a DBContext (Entity Framework). This is an implementation of IBackloadStorageProviderFile and can be used in your code.
ColumnMapping
Provides methods for mapping custom column names to column names used by the Backload Database plugin when executing SQL commands (usually chunked file uploads). Note: The Backload Database plugin uses the column names provided in the IBackloadStorageProviderFile interface by default. Column names can also be mapped in the configuration (Web.Backload.config). See demos.
FileStreamContext
Provides a database transaction context for SQL Server FILESTREAM operations.
ISqlChunkInfo

Interfaces
  InterfaceDescription
IBackloadMappedStorageProvider
Derived from IBackloadStorageProvider. Provides basic methods to be implemented by an external storage provider. This interface can be implemented by a DBContext (Entity Framework) or other classes providing a storage service. This interface can be used, when column names do not equal the default column names used by the IBackloadStorageProviderFile (custom data model). Adds a ColumnMapping property to map columns to the custom model.
IBackloadStorageProvider
Provides basic methods to be implemented by an external storage provider. This interface can be implemented by a DBContext (Entity Framework) or other classes providing a storage service.
IBackloadStorageProviderFile
