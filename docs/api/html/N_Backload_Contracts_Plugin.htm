Backload.Contracts.Plugin Namespace
This namespace contains definitions for supported client plugin handler interfaces. These interfaces are internally used to create a client plugin specific result object.
Interfaces
  InterfaceDescription
IClientPluginExtraChunkInfo
Used in client side plugins to provide additional chunks information.
IClientPluginExtraInfo
Used in client side plugins. Provides additional information usually not provided by the client plugin by default
IClientPluginExtraInfoEx
Used in client side plugins. IClientPluginExtraChunkInfo provides folder information to the client in the "extra" field for a file in the JSON response. To get this data returned it must be enabled in the configuration (Web.Backload.config) within the ReturnExtraInfo attribute.
IClientPluginFileItem
A default file item in client plugin handlers
IClientPluginFileItemCore
Basic file item in client plugin handlers
IClientPluginHandler
Provides a list of files processed in the current request for compatibility of derived client plugin handlers
IClientPluginResultCore
Provides core compatibility for client plugin handler results
Remarks
Only for internal use.
 Tip
It is strongly recommended to use the inferfaces provided in the Backload.Contracts namespace wherever possible instead of the implementations.
