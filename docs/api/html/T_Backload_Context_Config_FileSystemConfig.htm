AuxFilesFolder
Subfolder for supporting files (previews, mappings, etc). If "auxFilesFolder" is an empty string, supporting files will be stored in a subfolder within the upload folder. If "auxFilesFolder" is not empty (e.g. auxFilesFolder="_aux") supporting files will be stored in an external path ([filesRoot]\[auxFilesFolder]\). In this case the main upload directories only contain the uploaded files, which improves some processes like generating list of available files (GET) for large file lists. [Default: ""].

ObjectsRoot
Sub folder within FilesRoot. Unlike ObjectContext, which usually is provided by the client, ObjectsRoot is a server side property and will not posted back to the client.
Examples
You have several web applications which store files into different sub folders (ObjectsRoot) of FilesRoot. Another use case is, when you store files in user related folders, you can set ObjextsRoot to the user id. In this case the user uploads in his personal folder.
PathSeperator
Character used to separate path or directory levels. Default: '\'
VirtualDirectory
This property provides the name of a virtual directory in IIS or an absolute URL. If FilesRoot is relative to the web application root (starts with '~/', see filesRoot), you do not need to set this value. If FilesRoot is an absolute path, you usually set a virtual path in IIS to this external location. In this case, the virtualDirectory attribute must be set to the name of the IIS virtual path name and filesRoot to the absolute path. You can also set Virtual path the an absolute URL in order to set a root URL for downloads.
Examples
If the virtual path is z:\files and the name is "files", set filesRoot="z:\files" and virtualDirectory="/files" [Default: "/"].
